## Historik

| Version | Status | Hovedproblem | Hvorfor afvist/godkendt | Aendring i naeste version | Agent note |
|---|---|---|---|---|---|
| PF-v0 (tidligere) | Afvist | For vag og generisk formulering | Manglende operationel skarphed og uklar metodekobling | Indfoer tydelige begrebsdefinitioner, afgraensning og evidenskrav | Brug aldrig vage formuleringer uden målbar/fortolkbar forankring |
| PF-v1 (tidligere synopsis-varianter) | Afvist | Overfladiske delspoergsmaal | Ikke tilstraekkelig sammenhaeng mellem PF, teori og empiri | Indfoer claims-evidence matrix og anti-vaghed-regler | Undgaa at genbruge gamle PF-fraser ukritisk |
| PF-v2 (reeksamen-arbejde) | Aktiv | Skal gøres censor-robust | Under udbygning med teori_bibliotek og kontekstpakke | Laas PF foerst efter check mod `CLAIMS_EVIDENCE_MATRIX` | Agent skal validere hver PF-saetning mod kildehierarki |

## Godkendelseskriterier foer PF laases

1. PF er én klar saetning med tydelig afgraensning.
2. Hvert underspoergsmaal kan måles/fortolkes med defineret evidens.
3. PF er kongruent med videnskabsteoretisk og metodisk position.
4. PF kan besvares uden at gå uden for scope.

## Typiske fejl der ikke maa gentages

- Uklare kernetermer uden operationel definition.
- "Hvordan paavirker..." uden at definere indikatorer eller vurderingskriterier.
- Delsvar der ikke samlet besvarer PF direkte.
