# Synopsis Start Here

**Formål:** Dette er den primære indgang til repoet, når målet er at skrive eller kvalitetssikre en synopsis til videnskabsteori-eksamen.

## Læs i denne rækkefølge

1. `docs/VEJLEDER_FEEDBACK_SYNOPSIS.md`
2. `REGEL_OG_SKRIVEKOMPAS.md`
3. `AKADEMISK_RAMME.md`
4. `UDDANNELSE_OG_PENSUM.md`
5. `KVALITET_OG_SKRIVEGUIDE.md`

## Hvad filerne bruges til

| Fil | Rolle | Brug den til |
|---|---|---|
| `docs/VEJLEDER_FEEDBACK_SYNOPSIS.md` | **Primær eksamenslogik** | Vejlederens konkrete krav, fejlplaceringer og kildebrug i synopsis |
| `REGEL_OG_SKRIVEKOMPAS.md` | **Primær regelbog** | Operationelle regler, triggers, checks og standarder med ID'er |
| `AKADEMISK_RAMME.md` | **Primær faglig ramme** | Videnskabsteori, metode, teori og projektets overordnede logik |
| `UDDANNELSE_OG_PENSUM.md` | **Primær kildeprioritering** | Hvad der er Tier 1, og hvilke kilder der bruges til hvad |
| `KVALITET_OG_SKRIVEGUIDE.md` | **Primær skrivekvalitet** | Akademisk stil, struktur, tjeklister og censor-vinkel |

## Bindende synopsis-linje

- Scope skrives som **processen fra modtaget opgave til klientindstillet konsulent**.
- Kuada bruges til **korte begrebsdefinitioner, metode og kvalitetskriterier**.
- Rossman & Wilson bruges til **pragmatismestrategi, kombination af kvalitative/kvantitative logikker og triangulering som strategi**.
- Embedded case begrundes som **organisatorisk delmængde**, ikke workflow-states.
- Synopsis forsvarer **research design**, ikke bachelorens fulde metode-/analysekapitel.

## Sekundære støttefiler

- `docs/PENSUM_VIDENSBASE.md`
- `docs/CITATION_MAP.md`
- `docs/PENSUM_KILDEKORT_INDEX.md`
- `teori_bibliotek/`
- `docs/REPO_FILE_ROLES.md`

Disse filer bruges til uddybning, opslag og censorforberedelse, men de må ikke overtrumfe den bindende synopsis-linje ovenfor.

## Filer der ikke er primær sandhed

- `docs/10_HURTIGE_TRAENINGSLOG_LUKA.md` = personlig træningslog
- `videnskabsteori_arena/` = trænings-/quizlag
- `NotebookLM_Kilder/` = noter og study guides

De kan være nyttige, men de skal altid læses efter de primære filer.

## Hvis en fil siger noget andet

Brug denne prioritet:

1. `docs/VEJLEDER_FEEDBACK_SYNOPSIS.md`
2. `REGEL_OG_SKRIVEKOMPAS.md`
3. `AKADEMISK_RAMME.md`
4. `UDDANNELSE_OG_PENSUM.md`
5. øvrige filer

Hvis der stadig er konflikt, skal formuleringen markeres som usikker og rettes i repoet.
