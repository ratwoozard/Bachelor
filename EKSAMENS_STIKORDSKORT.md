# Eksamens stikordskort (1 side – print eller mobil)

**Kursus:** Theory of Science & Project Methodology · **Format:** 5 min fremlæggelse + 20 min spørgsmål  
**Pensumanker:** Holm (2023), Kuada (2012), Saunders (2023)

---

## 5 min – struktur (tjek mens du taler)

| Tid | Sig kort |
|-----|----------|
| 0:00 | Problem: spildtid / tempo i bemanding; **ikke** “virker AI generelt”. |
| 0:45 | **PF** én gang + scope: **SoluTalent** fra registreret/importeret opgave → **klientgodkendelse**; kun **loggede** trin kvantitativt. |
| 1:30 | **Pragmatisme** + **abduktion** + mini-eksempel (loops/review/override). |
| 2:40 | **Indlejret single-case**; data: **KPI + 4–6 interviews**; triangulering. |
| 3:40 | **C-T-D-C** + **insider** + negative indikatorer. |
| 4:40 | Output: flaskehalse, forklaringer, **forudsætninger** for færre manuelle trin. |

---

## C-T-D-C (én linje hver + handling)

| | Husk | Gør i casen |
|---|------|-------------|
| **C** Credibility | Troværdig tolkning for praksis | Triangulering; member check / validering af udsnit |
| **T** Transferability | Læser kan vurdere overførsel | Tyk kontekst: SS, scope, marked |
| **D** Dependability | Andre kan følge sporet | Audit trail: guide, kodning, dataudtræk |
| **C** Confirmability | Fund i data – ikke kun mavefornemmelse | Søg modbevis; vægt **override, loops, afvisninger** |

---

## Must-know (én sætning)

- **Pragmatisme:** Spørgsmålet driver metoden; viden = **nytte i casen**.  
- **Abduktion:** Observation ↔ forklaring; **ret** når kilder skilles.  
- **Analytisk gen.:** Til **teori/mekanismer** – ikke statistisk til alle firmaer.  
- **Triangulering:** Data-mønstre → interview-forklaring → krydstjek.

---

## 3 begrænsninger (altid sikre at nævne)

1. **Single-case** – ikke statistisk generalisering.  
2. **Logs** viser mønstre; **fuld kausalitet** kræver fortolkning.  
3. **Insider** – aktiv modvægt via negative KPI og dokumentation.

---

## 3 styrker

1. **To spor:** platform + interview.  
2. **Tydelig afgrænsning** af proces.  
3. **Kvalitet og bias** navngivet og håndteret.

---

## Ti spørgsmål – ét stikord svar hver

1. Pragmatisme? → begge datatype + anvendelighed  
2. Abduktion? → veksel + revision  
3. Case ikke survey? → dybde + workflow i kontekst  
4. Indlejret? → to enheder: flow + beslutning  
5. Analytisk gen.? → teori – ikke population  
6. Spildtid mål? → state, loops, override, reasons  
7. Uenige kilder? → abduktion / afvigelse som data  
8. Insider? → adgang vs normalisering + negative KPI  
9. Største grænse? → én case + periode + log ≠ alt  
10. Anvendelighed? → beslutningsgrundlag i SS-kontekst  

---

## Fallback hvis du går i stå

**Pragmatisme** → **SoluTalent** (tal + praksis) → **ikke universelle love**.  
**Abduktion** → **mønstre** → **forklaring** → **juster**.  
**Insider** → **triangulering** + **negative tal**.
