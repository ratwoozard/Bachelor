# AGENT_PLAYBOOK

**Hurtig start:** Se [`AGENT_QUICKSTART.md`](./AGENT_QUICKSTART.md) (10 linjer), derefter denne fil.

Formaal: Bindende arbejdsregler for AI-agenter i dette bachelorprojekt.

## 1) Scope og afgraensning

- Analyse scope er `staging_imported -> matched`.
- Jobsourcing, kontrakt og fakturering er out-of-scope.
- Teknisk fokus er funktionel analyse, ikke ML-udvikling.
- Pre-go-live praemis: ingen rigtige brugere og ingen produktionsdata i SoluTalent endnu.
- KPI/analytics omtales som maaleplan efter go-live, ikke som dokumenterede driftresultater.

## 2) Kildehierarki (bindende)

1. Tier 1: Holm (2023), Kuada (2012), Saunders (2023).
2. Tier 2: domaeneteorier (Lean, Davenport, TOE, DSS, Braun & Clarke).
3. Tier 3: klassikere til nuancering (fx Kuhn, Gergen, Gadamer, Popper).

Regel: Tier 3 maa aldrig erstatte Tier 1 i metode/videnskabsteori.

## 3) Outputkontrakt (obligatorisk format)

Hvert substanssvar skal have:
1. Paastand (hvad svares der paa)
2. Teori/kilde (forfatter + aar)
3. Evidenslogik (hvilken datakilde kraeves)
4. Begraensning (hvad svares der ikke paa)
5. Censor-vinkel (hvad kan kritiseres)

## 4) Anti-vaghed regler

Maa ikke:
- bruge generiske paastande uden kontekst.
- bruge kausal sikkerhed uden evidens.
- skrive "vi valgte X fordi det er bredt".
- generalisere statistisk fra case.

Skal:
- specificere begrebets funktion i jeres case.
- adskille maale-paastande fra fortolknings-paastande.
- markere usikkerhed og afgraensning.

## 5) Forbudte handlinger

- Ingen opfundne referencer.
- Ingen performance-paastande uden datahenvisning.
- Ingen brug af Kuada som primaer ontologi/epistemologi-fundament i denne ramme.
- Ingen formuleringer der antyder live driftmaaling, hvis evidensen kun er test/artefakt.

## 6) Standard evidenskrav

- Metodepaastande: Tier 1 + procedurebeskrivelse.
- Analysepaastande: mindst to empirikilder ved centrale fund.
- Konklusionspaastande: tydeligt skel mellem fund, fortolkning og implikation.

## 7) Bias-handelingsregel

Hvis paastanden vedroerer SoluTalent-kvalitet/effekt:
- inkluder potentiel bias ved naer caseadgang.
- inkluder mindst en alternativ forklaring.
